## 搬瓦工 KVM 支持哪些系统？先搞清楚底层逻辑

搬瓦工目前所有套餐都是 KVM 架构，OpenVZ 早已全线下架。KVM 的好处是对系统支持完整——你能跑的东西比 OpenVZ 多得多，内核可以自定义，Docker 跑起来也没问题。

KiwiVM 面板里目前支持一键安装的系统主要有这几类：

- **Debian**：11、12，64位为主
- **Ubuntu**：20.04、22.04、24.04，LTS 版本
- **CentOS**：7（还活着，但快到头了）、CentOS Stream 版本
- **AlmaLinux**：8、9，CentOS 的主要替代品之一
- **Rocky Linux**：8、9，另一个 CentOS 替代品
- **Fedora**：较新版本，滚动更快

这些系统对搬瓦工KVM装什么系统这个问题的影响，说白了就两个维度：**你打算用这台 VPS 干什么**，以及**你对 Linux 的熟悉程度**。

## 按用途推荐：不同场景装什么系统

### 场景一：新手学习 Linux / 跑脚本

装 **Ubuntu 22.04** 或者 **Ubuntu 24.04**。

原因很简单——资料多。遇到问题搜一下，99% 都能找到 Ubuntu 的解法。报错复制进搜索引擎，第一条结果通常就是答案。这对新手来说是最大的护城河。

内存占用方面，Ubuntu 的基础安装稍微比 Debian 多一点，但搬瓦工 KVM 最低配也有 1GB 内存，跑 Ubuntu 完全够用，不用纠结这点差距。

### 场景二：建站 / 挂宝塔面板

**Debian 12** 或者 **Ubuntu 22.04** 都行，我自己建站更倾向 Debian。

Debian 有个实际的优势：稳。它的软件版本更新比 Ubuntu 慢，但这在服务器场景下其实是好事——你不希望某次系统更新把你的站搞挂。装上宝塔之后跑个一两年，基本不会出幺蛾子。

内存占用也比 Ubuntu 小，同样配置下能多留一点给你的业务进程。

宝塔面板官方支持的系统列表里，Debian 和 Ubuntu 都在，两个都能装，不存在兼容性问题。

### 场景三：跑 Docker / 容器化部署

**Debian 12**，首选。

Docker 在 Debian 上运行非常顺畅，安装流程简单，官方文档直接就有 Debian 的安装脚本。内存基准线低，对小配置 VPS 比较友好。

Ubuntu 也完全可以，差异不大。如果你已经熟悉 Ubuntu 的操作方式，继续用 Ubuntu 就行。

### 场景四：企业应用 / 需要 RHEL 生态

装 **Rocky Linux 9** 或者 **AlmaLinux 9**。

CentOS 已经实际上死了——CentOS 7 在 2024 年就结束了生命周期，CentOS 8 更早。老用户死守 CentOS 的，迟早要迁移。Rocky Linux 和 AlmaLinux 都是 RHEL 的社区版，兼容性高，yum/dnf 那套操作逻辑一样，迁移成本最低。

如果你的项目是基于 RHEL 生态开发的，或者公司有企业 Linux 使用规范，就选这两个之一。两个选哪个差别不大，Rocky Linux 背后社区资源稍微活跃一点。

## 搬瓦工 KVM 重装系统教程（KiwiVM 面板操作）

重装系统是搬瓦工用户最常做的操作之一，步骤比你想象的简单。

1. **登录 KiwiVM 面板**：进官网后台，找到你的 VPS，点击"KiwiVM Control Panel"进入
2. **停止 VPS**：在"Main controls"里点"Stop"，等状态变为 stopped
3. **进入重装界面**：左侧菜单找"Install new OS"
4. **选择系统版本**：从下拉菜单选你要装的系统，推荐选最新的 64-bit 版本
5. **确认并安装**：勾选确认框，点击"Reload"，等待大约 2-5 分钟
6. **获取新密码**：重装完成后，面板会显示新的 root 密码，记录下来
7. **启动 VPS**：回到"Main controls"点"Start"，用 SSH 连接验证是否正常

注意：重装系统会**清空所有数据**，操作前务必做好备份。KiwiVM 面板里有快照功能，建议重装前先打一个快照留底。

搬瓦工 KVM 支持"DD 安装"的方式装 Windows，但这属于进阶操作，需要额外脚本，官方面板里没有 Windows 模板可以直接选。如果你的需求是 Linux 服务器场景，老老实实用上面几个 Linux 发行版就够了。

## 搬瓦工 KVM 装系统常见问题

**Q：搬瓦工支持装 Windows 吗？**

官方面板的系统列表里没有 Windows。如果一定要 Windows，需要通过 DD 安装的方式自己搞，有一定折腾成本，且稳定性不如 Linux 原生。大多数用搬瓦工的场景用 Linux 就能解决。

**Q：系统装好之后还能再换吗？**

随时可以换，KiwiVM 面板里就有重装入口，几分钟搞定。换系统会清数据，记得提前备份或者打快照。

**Q：CentOS 7 还能装吗？值得装吗？**

面板里还在，但 CentOS 7 已经停止了安全更新，不建议用于生产环境。新项目直接用 Rocky Linux 9 或 AlmaLinux 9 替代，维护成本更低。

**Q：Debian 和 Ubuntu 到底有什么区别，对新手来说重要吗？**

实际操作层面区别不大，包管理都是 apt，命令基本通用。主要差别是 Ubuntu 更新快、资料更新、新手更容易搜到解决方案；Debian 更稳定、更保守、适合长期跑的服务器。新手不知道选哪个就选 Ubuntu，用熟了之后换 Debian 也没什么学习成本。

**Q：内存只有 512MB 或 1GB，哪个系统更省资源？**

Debian 基础安装占用内存最小，适合小内存机器。搬瓦工 KVM 基础套餐最低 1GB 内存，装 Debian 12 剩余内存给业务用还比较宽松；装 Ubuntu 也够用，但相对会稍微紧一点。

**Q：重装系统会不会丢失 IP？**

不会。重装只是换操作系统，IP 地址不变，SSH 端口也不变。改变的只是系统内的所有数据和配置。
